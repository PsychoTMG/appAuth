# Full-Stack appAuth
Современная полностековая система управления, построенная на базе клиент-серверной архитектуры с акцентом на безопасность, масштабируемость и интерактивный пользовательский интерфейс.

## Архитектура и Стек технологий
Проект разделен на два независимых приложения: Frontend (Next.js) и Backend (NestJS), которые обмениваются данными через REST API.

### Frontend
* **Фреймворк:** Next.js
* **Стилизация:** Tailwind CSS
* **HTTP-клиент:** Axios
* **Управление состоянием:** React Context API
* **UI/UX:** Кастомные анимации (печатный текст, интерактивные цветовые сетки), `react-hot-toast` для уведомлений.

### Backend
* **Фреймворк:** NestJS
* **База данных:** PostgreSQL
* **ORM:** Prisma
* **Аутентификация:** JWT, Passport.js, Bcrypt
* **Конфигурация:** `@nestjs/config` для безопасной работы с переменными окружения.

## Безопасность и Авторизация
В проекте реализован безопасный флоу аутентификации, защищающий от XSS и CSRF атак:
1. **Access Token:** Возвращается в теле ответа при логине/регистрации и хранится в памяти приложения (Context API) на клиенте.
2. **Refresh Token:** Устанавливается сервером исключительно в виде `httpOnly`, `Secure` (в production) куки. Клиентский JS не имеет к нему доступа.
3. **Ротация:** При истечении Access токена, клиент обращается к эндпоинту `/auth/refresh`, сервер валидирует куку и выдает новую пару токенов.
4. **Хеширование:** Пароли пользователей необратимо хешируются с помощью `bcrypt` перед сохранением в БД.

## Локальный запуск
### 1. Подготовка базы данных
Убедитесь, что у вас установлен PostgreSQL. Создайте пустую базу данных для проекта.
### 2. Запуск Backend (NestJS)
Перейдите в директорию бэкенда:
```bash
cd backend
npm install
```

Создайте файл `.env` в корне бэкенда и добавьте необходимые переменные:
```env
DATABASE_URL="postgresql://USER:PASSWORD@localhost:5432/YOUR_DB_NAME?schema=public"
PORT=3001
JWT_SECRET="your_super_secret_key"
JWT_ACCESS_TTL="15m"
JWT_REFRESH_TTL="7d"
COOKIE_DOMAIN="localhost"
NODE_ENV="development"
```

Примените миграции Prisma и запустите сервер:
```bash
npx prisma db push
npm run start:dev
```
Сервер будет доступен по адресу: `http://localhost:3001`

### 3. Запуск Frontend
Перейдите в директорию фронтенда:
```bash
cd frontend
npm install
```
*(Опционально)* Создайте файл `.env.local` для выноса API URL:
```env
NEXT_PUBLIC_API_URL="http://localhost:3001"
```

Запустите клиентское приложение:
```bash
npm run dev
```

Клиент будет доступен по адресу: `http://localhost:3000`

## Структура проекта
* **Frontend:** Использование Server и Client компонентов. Выделенные Layouts для изолирования логики отображения хедера и страниц авторизации.
* **Backend:** Модульная архитектура NestJS. Выделены `AuthModule` и `PrismaModule`. Использование Guards (`JwtAuthGuard`) для защиты приватных эндпоинтов (например, `/auth/profile`).
